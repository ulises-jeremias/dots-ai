# User Story (WHAT)

Use for user-centered requirements that need business value, technical notes, and acceptance criteria.

## Default guardrails

1. Apply **`dots-ai-output-handshake`** before final output.
2. Use **`dots-ai-planning`** when estimation or splitting is needed.
3. Use **`dots-ai-task`** for purely technical work that is not a user story.
