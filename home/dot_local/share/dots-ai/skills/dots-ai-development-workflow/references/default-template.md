# Default Development Workflow Checklist

## Scope

Use only when the project has no explicit workflow override.

## Status Lifecycle

- [ ] Backlog: item exists but is not yet estimated/scoped/planned.
- [ ] Ready TODO / TODO: scope, AC, dependencies, and estimate are ready.
- [ ] In Progress: development and local testing started.
- [ ] Blocked: blocker, impacted environment, and dependency owner documented.
- [ ] Ready for Review: PR/MR created, linked, checks passing or documented.
- [ ] Ready for Acceptance: testing notes, evidence, and AC validation ready.
- [ ] Ready for QA: QA validation can start.
- [ ] Ready for Release: final checks completed.
- [ ] Closed: released/done/not released status selected with final validation notes.

## Definition of Ready

- [ ] Clear description and scope.
- [ ] Acceptance criteria agreed.
- [ ] Dependencies and impact documented.
- [ ] Estimate and rationale captured.
- [ ] Designs / technical alignment completed if applicable.
- [ ] Owner and collaborators known.

## Definition of Done

- [ ] Code follows project standards.
- [ ] Peer review completed.
- [ ] CI/CD checks pass or exceptions are documented.
- [ ] Automated/manual validation completed as needed.
- [ ] Acceptance criteria verified.
- [ ] Documentation and communication updated.
- [ ] Temporary branches/flags cleaned up when appropriate.

## Evidence and Traceability

- [ ] Work item linked to PR/MR.
- [ ] Test steps and results documented.
- [ ] Screenshots/recordings attached when useful.
- [ ] Deployment or release evidence linked when applicable.
- [ ] Traceability exists: PRD -> Task -> PR/MR -> Deployment.
